Mathematical MDP Formulation

AIR-NDF models the network discovery process as a finite Markov Decision Process (MDP) represented by the tuple (S, A, P, R, γ):

1. Observation State Space (S ∈ R24)

To allow feedforward neural networks (MLPs) to process dynamic graphs without computational overhead, the network state is encoded as a standardized 24-dimensional feature vector:

  • Local Host Signatures (Index 0-4): Encodes current node ID, host classification, vulnerability score, detection risk, and reward.
  • Global Topology Discovery Stats (Index 5-8): Percentage of overall nodes discovered, percentage explored, cumulative risk, and step budget remaining.
  • Subnet & Cluster Frontiers (Index 9-16): Cardinality of adjacent unexplored nodes, active subnet exploration density, active NetworkX community cluster metrics.
  • Vulnerability & Reward Profiles (Index 17-23): Expected value (mean/max) of vulnerabilities and rewards within adjacent, subnet, and cluster candidate lists.

2. Macro-Exploration Action Space (A)

The agent selects from 5 high-level heuristic exploration policies:

Action = {
  0: Explore Adjacent Node (local edge expansion),
  1: Jump to Inferred Node (cross-subnet connection exploration),
  2: Prioritize Subnet (intra-subnet scan),
  3: Investigate Cluster (community/highly-connected mesh scan),
  4: Terminate Discovery (safe exit strategy to avoid detection)
}

3. Dynamic Reward Shaping Function (R)

The agent's objective function balances information gain against discovery cost and detection penalty:

R = Rdiscovery(vulnerability, value) - Cstep - Crisk(detection_probability) - Credundant

If a simulated detection event is triggered based on the node's detection_probability, the episode terminates immediately with a significant penalty (-300). Self-termination (Action 4) rewards the agent with a bonus proportional to the remaining threat safety margin, incentivizing risk mitigation.
